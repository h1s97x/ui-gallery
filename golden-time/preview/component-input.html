Summary

Component guidance

Use the input component for short to medium text entry. The field should feel calm and highly legible, with low visual noise at rest and strong clarity on focus or validation.

  • Labels should name the data directly: Search clients, Billing email, Filter by region.
  • Helper text should explain format or consequence, not repeat the label.
  • Error text should be specific enough to resolve in one pass.
Accessibility

Every input requires a visible label or programmatically associated accessible name, and error text should connect through aria-describedby.

Behavior

Leading icons support search and lookup, while trailing actions such as Clear stay textually explicit when space allows.
