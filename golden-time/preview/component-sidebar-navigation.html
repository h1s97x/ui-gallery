Summary

Component guidance

Use sidebar navigation for desktop and large-tablet layouts where users need a stable map of destinations. The active item should feel persistent, not like a temporary hover treatment, and the overall surface should stay quieter than dashboard cards.

  • Use short nouns for destinations: Overview, Members, Billing, Reports, Settings.
  • Badges should highlight counts or plan context only when the information changes navigation priority.
  • Warm accent states make the navigation tactile and elegant without introducing new color families.
Accessibility

Use nav and list semantics, keep every target keyboard-reachable, and apply aria-current="page" on the selected destination.

Behavior

Collapsed sidebars require tooltips or another reveal pattern so icons stay unambiguous when labels are hidden.

Composition

Dashboard placement guidance

The sidebar should support movement and orientation while the data layer stays center stage.

Stable desktop movement

Use the expanded variant on desktop dashboards where users frequently move between Overview, Members, Billing, reports, and settings.

Information-dense workspaces

Use the sectioned variant when destinations need editorial chapters that reflect how people think about the product.

Compressed layouts

Use the compact or collapsed pattern only when the product also provides tooltip or reveal support for hidden labels.
