Summary

Component guidance

Use the data table for dense comparison, sorting, and review workflows. The component should remain highly legible and structured, with restrained row treatments so numerical patterns stay easier to scan than decorative chrome.

  • Prefer business-facing labels such as Client, Amount, Renewal date, and Status.
  • Keep row actions consistent so operational teams build scanning memory across views.
  • Use muted supporting text for metadata before adding more columns.
Layout Right-align quantitative columns and left-align descriptive content for quicker scanning.
Accessibility Use native table semantics and expose the active sort state with aria-sort.

Dashboard placement guidance

Tables often live inside cards, but can also become the main page artifact when the report is the task.

Invoice review

Use the standard variant for Open invoices where sorting, metadata, and actions need equal clarity.

Bulk operations

Use the selectable variant for approval, archive review, or ownership changes that require selection counts.

Chart adjacency

Use the compact variant beside a Revenue trend panel or narrative summary when the dashboard must stay dense.
