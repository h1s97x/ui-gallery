Feature-specific pricing

Scan modifiers for cache, residency, fast mode,
batch jobs, context, and tools.

Cache

Prompt caching

  • 5m write: 1.25x base input.
  • 1h write: 2x base input.
  • Cache hit: 0.1x base input.
Residency

Data residency

Supported first-party requests with US-only inference carry a 1.1x multiplier across token categories.

Preview

Fast mode

TraeWork preview fast mode applies premium input and output rates for supported N-series versions and is not available with Batch API.

Discount

Batch processing

Batch API applies a 50% discount to input and output token rates for asynchronous workloads.

Context

Long context

The TraeWork reference states that selected N-series, P-series, and R-series versions include the full 1M token window at standard pricing.

Tools

Tool use

Tool use includes normal input and output tokens, tool definition overhead, tool call/result content, and server-side tool charges where applicable.

Additional considerations

Use these notes to control cost, capacity,
discounts, and billing setup.

Cost

Optimization strategies

  • Match model tier to task complexity.
  • Use prompt caching for repeated context.
  • Use Batch API for non-urgent volume work.
  • Watch usage patterns by tokens and tool calls.
Limits

Rate limits

Usage tiers affect request capacity. The TraeWork reference lists entry, growth, established, maximum standard, and enterprise paths.

Discounts

Volume and enterprise

High-volume, enterprise, academic, and research arrangements may use negotiated discounts or custom terms.

Billing

Payment basics

Billing is based on monthly usage, paid in USD, with card and invoicing options depending on account type.

Frequently asked questions

Quick answers for token accounting,
cache discounts, and payment setup.

How is token usage calculated? +

Tokens are text units processed by a model. The exact count depends on language, content type, and tokenizer version.

Can prompt caching and Batch API discounts stack? +

The TraeWork reference says they can be combined, with caching multipliers and batch discounts applied to the relevant token categories.

What payment methods are supported? +

Standard accounts can use major credit cards, while enterprise customers can arrange invoicing or negotiated billing terms.
